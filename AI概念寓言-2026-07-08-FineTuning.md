## 寓言故事

### 一、答对了，却像「别人家的人」

Grounding 上线一个月，客服抽检的「引用准确率」稳在 82%。林薇刚把周报发给老板，销售总监老赵就推门进来，把手机往桌上一放。

「你们自己看这条。」

用户问：「**旗舰版** 能不能和 **飞书** 做审批流对接？」

小言回复工整、脚注齐全：

> 支持。请管理员进入 **管理后台 → 集成中心 → 第三方审批**，按向导完成配置。[1]

老赵指着屏幕：「**事实没错**，脚注我也点开了——但用户问的是 **我们叫了一年的『星舰版』**，小言回的是 **『旗舰版』**；我们对接的是 **企业微信审批**，它写 **飞书**。法务要的是出处，销售要的是 **一听就是我们的人**。」

运营同学辩解：「通用大模型训练数据里，『旗舰版』比『星舰版』常见……检索段里其实写了星舰版，它一组织句子又滑回公网说法了。」

老赵叹气：「知识库能救 **对不对**，救不了 **像不像**。用户会觉得：你们机器人是不是 **外包的**？」

散会后，老周在白板上画了三格：

```
Prompt（便签）  →  每次对话临时叮嘱
RAG（手册）    →  当场翻资料找依据
???            →  把「本公司口音」写进本能
```

「第三格，」老周说，「就是今天要算账的那件事。」

---

### 二、便签贴满桌，手册翻烂了

林薇先走老路：**加厚 Prompt**。

她在系统指令里加了一页「话术禁忌」：必须使用 **星舰版 / 企业微信审批 / 工单号 WO-** 前缀；禁止用竞品名替代已对接产品；回复结构固定为「结论 → 步骤 → 引用」。

效果立竿见影——**测试集上** 像了八成。

但灰度到真实用户第三天，问题冒出来：

**第一，便签越贴越长。** 话术规范、格式模板、禁词表、示例对话……光系统 Prompt 就吃掉大几千 **Token（词元）**——圆桌本就不宽，留给用户问题和检索段的空间被挤扁。

**第二，用户一变招就破功。** 有人问「**顶配那档** 行不行」，小言没认出指的就是星舰版，又滑回「旗舰版」——**规则列的是词表，用户说的是口语**。

**第三，维护像叠补丁。** 销售每周改一版对外称谓，林薇就得改 Prompt、回归测试、重新发版。小陈吐槽：「像在实习生脑门上 **每天贴一张新便签**，贴到便签比脸大。」

RAG 侧也试过：把《对外称谓白皮书》整本塞进知识库，检索能捞到「星舰版 = 原旗舰版更名」。可小言 **读到了**，组织语言时仍习惯性写「旗舰版」——像背过书的学生，一紧张又冒出 **网上最常见的说法**。

老周总结：「**手册解决『知不知道』，便签解决『这次别忘了』**——但 **说话习惯、肌肉记忆、默认措辞**，不是翻手册能彻底改掉的。」

---

### 三、口音班招生简章

算法小陈在评审会上摊开一份方案，标题写着：**「星舰口音计划 —— 领域微调」**。

「通俗讲，」小陈说，「不是每天贴便签，而是 **把小言送回培训班**，用咱们自己的 **金牌客服对话记录**，练一段时间，让它 **默认就说星舰版、就说企业微信**——行业里叫 **Fine-tuning（微调）**：在通用大模型已经读过『全世界书』的基础上，用 **本公司教材** 再训一轮，把 **权重** 往咱们的说话方式上拧一拧。」

他画了张简图：

```
通用大模型（已预训练）
        ↓
  筛选 / 清洗 / 标注
  历史优质对话、工单、话术样本
        ↓
  微调训练（算力 + 时间 + 调参）
        ↓
  评测 → 上线「星舰专属小言 v1」
```

销售老赵眼睛一亮：「要多久？多少钱？」

小陈如实报账：**数据清洗两周、训练跑数天、评测一轮一周**；算力账单按 **GPU 小时** 计；更麻烦的是——**每次大版本升级，可能要重新微调或至少回归一轮**。

林薇问关键问题：「微调以后，**产品改名** 怎么办？**新功能** 文档还没进训练集，它会不会 **瞎编得很自信**？」

小陈点头：「所以微调适合 **相对稳定** 的东西：**口吻、格式、行业惯用表达、常见意图的处理套路**；**天天变的价目表、新上的接口**，仍然应该靠 **RAG + Grounding**，不能写进骨子里。」

老周补了一句：「像送员工去 **三个月口音与礼仪培训**：出来 **默认更像本公司**；但 **具体报价、今天是否维护**，还得 **每天看公告板**。」

---

### 四、培训班里的三堂事故课

试点数据来自去年 **TOP 客服** 的 1.2 万条对话。训练进行到一半，质检连报三堂课——林薇把它们写进事故复盘：

**事故 A：学到了坏习惯。**  
训练集里混进一批「为了催单口头承诺加急」的旧对话。微调后的小言 **礼貌得更像销售**，却会在没有权限依据时脱口而出「**帮您特殊申请加急**」——**口音对了，合规错了**。  
教训：**微调数据 = 新员工的教材，脏教材出脏本能**；必须 **清洗、分级、合规抽检**，不能「有多少对话就喂多少」。

**事故 B：学会新词，忘了旧常识。**  
过度强调「星舰版」后，测试里出现 **通用能力轻微退化**：个别常识类闲聊答得变短、变刻板；还有一条把 **旧产品名** 当成现行名——像员工培训后 **只会新口号，偶尔忘了老产品还在服** 。  
教训：**微调是改权重，不是纯加法**；要设 **通用能力回归测试**，防止 **捡了口音丢智商**（产品上要准备 **回滚基线模型**）。

**事故 C：把事实焊死在嘴里。**  
有人提议把最新 **价格表** 也塞进训练集，一劳永逸。小陈坚决反对：「价目 **下月就改**。写进权重里，改起来要 **重新训 + 重新发版**；RAG 改一条文档就行。把 **高频变的事实** 微调进去，是 **用焊枪修公告板**。」  
教训：**微调擅长大方向习惯，不擅长高频变更的事实**；事实仍走 **知识库 + 引用**。

三轮清洗、一版保守学习率、加上 **通用测试集** 对齐后，「星舰小言 v1」再进灰度。

同一条飞书审批的问题，它答：

> **星舰版** 支持对接 **企业微信审批**。[1]  
> 请管理员进入 **星舰控制台 → 生态集成 → 企业微信审批**，按向导完成配置。[1]

脚注仍在，但 **称谓、产品路径、默认措辞** 都贴上了自家商标感。老赵只提了一个小意见：「『生态集成』是我们文档里的栏目名，它终于 **不瞎翻译成集成中心了**。」

### 五、算账：什么时候值得交这笔学费

全量上线前，林薇在 PRD 里写了一张 **决策表**，会上吵了一轮，最后达成共识：

| 需求类型 | 优先方案 | 理由 |
|----------|----------|------|
| 公司专属称谓、口吻、回复结构 | **微调** | 稳定、高频、靠 Prompt 易漏 |
| 价格、政策、接口、是否支持 | **RAG + Grounding** | 变更频、要出处、可热更新 |
| 临时活动话术、A/B 文案 | **Prompt** | 迭代快、成本低 |
| 必须实时查库存 / 工单状态 | **Tool Calling** | 事实在外部系统，不在模型肚子里 |

老周在上线邮件里写：**「微调不是银弹，是 **口音与礼仪班**；手册和便签照样要——班只解决 **像不像我们的人**，不解决 **今天公告写了啥**。」**

陈姐补了一条合规：**微调训练集视同内部数据资产**——脱敏、授权、禁喂用户隐私；版本号进审计日志，和模型权重一起 **可回滚、可追责**。

灰度两周，「不像自家产品」相关投诉 **下降 34%**；Token 成本也因系统 Prompt 瘦身 **降了一截**。但林薇在复盘里记了一句红字：**「下次改名星舰版 v2，预算里要预留 **重新微调 or 强化 RAG** 的排期。」**

散会上，新人问：「所以微调到底是啥？」

林薇答：「不是让模型 **多读几本公司 PDF**，而是 **改它默认怎么说话**——学费贵、周期长，适合 **值得写进肌肉记忆的东西**；其余交给 **手册和便签**。」

## 概念正解

**Fine-tuning（微调）** 指：在已经训练好的通用大模型基础上，用 **特定领域或业务的高质量样本** 进行 **额外训练**，调整模型内部参数，使其在 **口吻、术语、格式、常见意图处理** 等方面更贴近你的场景；它解决的是「 **每次都要靠 Prompt 反复强调，仍容易滑回公网默认说法** 」的 **习惯层** 问题，**不是** 用来替代 **频繁变更的事实** 或 **实时外部数据**。

回扣故事：

- **「星舰版被写成旗舰版」** → 通用模型的 **默认措辞** 与业务 **专属表达** 冲突，单靠 RAG「知道」不够；
- **「便签贴满桌」** → Prompt 能临时约束，但 **长、脆、难维护**；
- **「送回培训班」** → 用 **历史优质对话** 做额外训练，把偏好 **写进权重**；
- **「脏教材、忘常识、把价目焊死」** → 数据质量、灾难性遗忘、事实与习惯 **分工错误** 三大坑；
- **决策表** → 微调与 RAG / Prompt / Tool Calling **组合使用**，各管一层。

在 **AI 产品岗** 的典型场景：

| 场景 | 你要怎么用这个概念 |
|------|-------------------|
| **需求文档** | 写清 **微调目标**（口吻 / 术语 / 格式 / 意图路由），**训练数据来源与清洗规则**，**不微调清单**（价格、政策等走 RAG），**版本与回滚策略** |
| **方案评审** | 问算法：「数据量够不够？有无 **通用能力回归集**？学习率保守吗？上线后 **如何与 RAG 分工**？基座模型升级怎么办？」 |
| **指标设计** | 除 **术语准确率、口吻一致性** 外，监控 **通用问答退化、幻觉率是否上升、微调后 Token/Prompt 成本变化**；事实类仍看 **引用准确率** |
| **跨团队沟通** | 跟业务说：「微调是 **口音班**，不是 **万能知识灌输**」；跟法务说：「训练集要 **脱敏与授权**」；跟研发说：「改名、大促、接口变更 **别只指望重新训**」 |

## 类比锚定

**核心类比：新员工入职后的「企业文化与岗位特训营」——先招一个通才（通用模型），再送他去 **内部集训** 学公司称谓、话术结构、服务礼仪；但 **每日价目、当天排班、实时库存** 仍查 **内部系统 / 公告板**，不能指望集训背下一切**

传统 PM 多半组织过 **入职培训、话术认证、销售礼仪班**：通才来了先懂行话、懂流程、懂「我们怎么称呼客户」，这是 **肌肉记忆**；具体报价、今日是否维护，仍看 **CRM / 知识库**。Fine-tuning 对应 **集训改本能**；RAG 对应 **上岗后翻手册**；Prompt 对应 **班组长每天晨会强调的注意事项**。

| 你熟悉的 PM 经验 | Fine-tuning 要素 |
|------------------|------------------|
| 入职集训：学公司简称、客户称谓、标准问候语 | 用 **领域对话样本** 训练 **默认术语与口吻** |
| 集训教材由 HR / 金牌员工编写、审核、淘汰过时案例 | **训练数据清洗、合规抽检、去脏样本** |
| 集训周期数周、占用讲师与场地（一次性成本） | **GPU 算力、标注、调参、评测** 的 **前期投入** |
| 集训只教「怎么说像自己人」，不背每日特价 | **习惯层微调** vs **事实层 RAG** 分工 |
| 产品大改版后要 **复训** 或发 **补充手册** | 基座升级 / 品牌更名 → **重新微调或强化检索** |
| 培训过度可能把新人训得只会话术、不会变通 | **过拟合 / 通用能力退化**，需 **回归测试与回滚** |

## 类比边界说明

### 一、概念核心要点（回扣定义）

1. **微调改的是模型的「默认倾向」**（怎么说、用什么词、什么结构），不是 **可靠记忆可变事实** 的最佳手段。
2. **训练数据质量决定上限**：脏数据、违规承诺、过时说法会被 **放大成系统本能**，清洗与合规与算法同等重要。
3. **存在成本与周期**：数据准备、算力、评测、版本管理；**基座模型升级** 常触发 **再训练或重测**，不是一锤子买卖。
4. **要与 RAG、Prompt、Tool Calling 组合**：微调 **口音**；RAG **事实与出处**；Prompt **短期试验**；工具 **实时外部状态**。
5. **需防范通用能力退化与过拟合**：要有 **独立测试集、回滚方案**，避免「像了」但「笨了」或「只会训练集里见过的题」。

### 二、这个类比在哪里会失效？

| 类比说法 | 失效点 | 更准确的理解 |
|----------|--------|--------------|
| 「像集训一样，训完就一劳永逸」 | 业务 **称谓、政策、接口** 会变；权重里的旧事实 **不易热更新** | 事实走 **RAG**；微调目标锁定 **相对稳定** 的表达与格式 |
| 「教材越多越好」 | 低质、重复、过时对话 **污染权重**；小样本也可能 **过拟合** | 重 **质** 与 **代表性**；设数据准入与 **held-out 评测** |
| 「集训比晨会便签高级，所以优先上」 | 微调 **迭代慢、试错贵**；很多需求 **改 Prompt 半小时能测** | 先用 **Prompt / RAG** 验证需求是否真实；微调留给 **反复出现、稳定、值得固化** 的模式 |
| 「训成自己人就不会乱说」 | 微调 **不保证事实正确**，甚至可能 **更自信地错** | 事实类仍要 **Grounding**；微调后 **幻觉率** 也要监控 |
| 「每个客户都该有一个专属模型」 | 多租户各自微调 → **运维、成本、版本爆炸** | 评估 **按租户微调 vs 统一基座 + 租户 RAG/Prompt**；多数场景 **不必人人一班** |

### Fine-tuning 产品 Checklist（可保存到笔记）

```markdown
## 值不值得微调
- [ ] 问题是「像不像 / 默认怎么说」，不是「知不知道最新事实」
- [ ] 规则已用 Prompt 试过，仍高频失败且规则稳定
- [ ] 业务接受：迭代周期周级、非小时级

## 数据与合规
- [ ] 训练集来源：金牌对话 / 标注样本，非 raw 全量日志
- [ ] 脱敏、用户授权、禁止违规承诺进入样本
- [ ] 脏数据、过时政策、竞品误导案例已剔除

## 训练与评测
- [ ] 目标：术语、口吻、结构、意图 —— 写进 PRD
- [ ] 通用能力回归集（闲聊、常识、安全）不过线
- [ ] 事实类仍走 RAG + Grounding，单独测引用准确率

## 上线与运维
- [ ] 模型版本号、训练数据版本、可回滚基线
- [ ] 基座升级 / 品牌更名的再训练预算
- [ ] 监控：口吻一致性、幻觉率、用户「不像自家」投诉
```
