## 寓言故事

### 一、财务问：小言这个月为什么贵了三倍？

月初第一个工作日，财务苏姐把一张表投进产品群，@林薇：

「AI 推理费 **上月 8 万，本月 24 万**——用户量只涨了 **40%**，费用怎么 **翻了三倍**？老板问能不能 **下个月压回 10 万**。」

群里安静了半分钟。

销售老赵说：「说明 **用得好啊**！客诉少了，续费多了—— **ROI 肯定划算**。」

苏姐回：「ROI 要 **算得出来** 才行。你们 PRD 里写的 **单次会话成本 0.3 元**，现在财务摊下来 **快 1.2 元**—— **差四倍**。」

林薇心里一沉：Demo 时大家盯的是 **回答漂不漂亮**；评测上线后盯的是 **准不准、稳不稳**——**没人持续盯「每答一次花多少钱」**。

她回复苏姐：「给我 **两天**，我拆一条真实会话，看钱 **花在哪**。」

---

### 二、一条会话，十二趟「运费」

林薇拉上小陈和大刘，从账单系统里 **随机抽了十条** 上月高成本会话——都是 **客服小言** 场景。

他们盯住其中一条 **看起来很普通** 的工单：用户问「企业版续费能不能开发票、抬头怎么改」。

大刘把 **调用链日志** 投屏。林薇原以为：**用户问一句 → 模型答一句**，顶多两笔账。

实际链路像 **快递员在同一条街上跑了十二趟**：

```
第 1 趟：意图识别（小模型）          → 判定「续费 + 发票」
第 2 趟：检索知识库 Embedding        → 把用户问句变成向量
第 3 趟：Rerank                      → 三十条候选压成三条
第 4 趟：主模型生成（旗舰版）        → 读了 3 条手册 + 12 轮历史
第 5 趟：用户追问「抬头用旧公司名行不行」
第 6 趟：Agent 决定再查 CRM          → Tool 返回一整页客户档案
第 7 趟：主模型再次生成（旗舰版）    → 上下文里塞进 CRM 大 JSON
第 8 趟：用户说「那帮我提交工单」
第 9 趟：Agent 调 create_ticket
第 10 趟：主模型写确认话术（旗舰版）
…… 还有 Guardrails 复检、日志摘要各一趟
```

小陈在白板旁写：

```
一次「感觉很简单」的续费咨询
= 多次模型调用 + 越来越厚的上下文
= 运费 × 重量 × 趟数
```

林薇问：「 **旗舰版** 为什么全程开着？」

小陈苦笑：「上线前怕 **质量不稳**，路由全切 **最大模型**—— **省事**。」

大刘补刀：「CRM 返回 **没裁剪**，一次 Tool 就 **塞了四千 Token**；知识库检索默认 **Top10**，其实 **Top3 就够**。」

林薇忽然想起 Token 那篇里老周说的：**桌上只能摞这么厚**——现在不是纸不够，是 **每次都搬整库上楼，还雇最大号卡车**。

---

### 三、不是「少说话」，是「分包裹、选车型」

第二天晨会，林薇没谈「砍功能」，她画了一张 **运费站分拣图**：

**第一刀：分车型——模型路由（Model Routing）。**

「 **开发票规则** 这种 FAQ，用 **小模型 + 手册引用** 就够； **合同解释、报价谈判** 才升 **旗舰版**。」

小陈展示试点数据：同样一百条金标集， **FAQ 类** 用小模型， **事实准确率只掉 1%**，单次成本 **降 70%**。

**第二刀：减重量——上下文与检索瘦身。**

- Tool 返回 **只留字段**：抬头、税号、订单号—— **别整页 JSON 上桌**；  
- 检索 **Top10 改 Top3**，配合 Rerank 已够用；  
- 历史对话 **摘要进上下文**，别 **逐字复读十二轮**。

阿寻举手：「手册分块 **切细了** 也会贵—— **一块半页纸**，检索命中三块， **输入 Token 照样爆**。」

**第三刀：同一路线别跑两趟——缓存（Caching）。**

大刘说：「 **系统 Prompt、热门 FAQ、固定政策段** 可以 **缓存 Prompt 前缀**——同样开头， **第二次起半价**； **完全相同的检索问法**， **Embedding 结果也能短缓存**。」

**第四刀：别空车返程——批处理与异步。**

离线 **评测、日志打标、批量摘要** 别挤在白天 **和在线抢同一批 GPU**；能 **合并请求** 的报表任务 **走夜间批处理价**。

老周听完，只提醒一句：「降本 **别偷质量**—— **哪类场景降了、指标有没有退**，要挂回 **评测尺子** 上。」

---

### 四、产品表上，多了一列「单次成本」

一周后，林薇在 PRD 模板里 **加了一栏**，同事起初嫌烦，月底却安静了：

| 功能点 | 预期单次成本 | 主要开销来源 | 降级/兜底策略 |
|--------|--------------|--------------|---------------|
| FAQ 发票 | ≤ 0.08 元 | 小模型 + Top3 检索 | 超预算则 **禁开旗舰** |
| 续费谈判 | ≤ 0.45 元 | 旗舰 + CRM 裁剪字段 | 超 3 轮 **转人工** |
| 夜间批标注 | ≤ 0.02 条 | 批处理 API | 与在线 **分池** |

他们在看板里加了 **三条产品指标**，和财务对齐：

1. **单次会话推理成本（元）**——按场景分层，不看笼统平均；  
2. **每解决一单有效咨询的成本**——和 **人工坐席成本** 比，不算自嗨；  
3. **成本异常率**——超过阈值会话 **自动采样**，查 **是否 Agent 死循环、Tool 爆量**。

试点两周后，苏姐发来消息：「总账 **从 24 万降到 15 万**；续费场景 **满意度没掉**。老板问： **还能不能再压？**」

林薇答：「能，但下一刀不是 **换更便宜的模型**，而是 **产品策略**——例如 **免费用户每日深度对话上限**、**复杂工单引导自助**、**高峰期限流**。 **成本优化是持续运营**，不是上线前改一行配置。」

---

### 五、最贵的坑：Demo 免费，量产按斤卖

复盘会上，林薇把踩过的坑贴墙上——她这时才明说，故事里一直在讲的 **「运费」**，就是 **推理成本**：

**坑 1：只测效果，不测账单。**  
Demo 用 **旗舰 + 全量上下文**；上线后 **流量一来就爆**——像 **试吃免费、外卖按克计费**。

**坑 2：Agent 越聪明，趟数越多。**  
每多一轮 **「我再查一下」**，都是 **新一轮运费**；死循环不熔断， **一晚能烧掉一台车的钱**。

**坑 3：降本偷跑质量。**  
偷偷全局换 **超小模型**，评测 **只跑简单题**——月底钱少了， **投诉多了**， **更贵**。

**坑 4：财务口径和产品口径不一致。**  
PRD 写 **0.3 元**，没含 **Embedding、Rerank、复检、重试**—— **全链路成本** 才是一单真相。

**坑 5：不做分层，全员顶配。**  
像 **寄明信片也用空运**——产品要敢写：**谁用什么车型、什么场景必须升舱**。

小陈在文档里给这套东西起名：**Inference Cost Optimization（推理成本优化）**——从 **模型路由、上下文治理、缓存、批处理、限额策略到成本看板** 的一整套 **产品 + 工程协同**。

林薇跟新人说：「 **Token 是计价单位**， **推理成本是经营问题**。会设计 AI 功能，还要会设计 **花得起、扩得开** 的 AI 功能。」

---

## 概念正解

**Inference Cost / 推理成本优化**，是指在 **保证可接受质量与安全底线** 的前提下，管理并降低 AI 系统 **每次推理（生成、检索、工具编排、复检等）** 所消耗的计算与 API 费用的一整套产品与技术手段。它解决的核心问题不是「让模型少说两句」，而是：**在用户量、链路复杂度、模型单价三者乘积爆炸之前，用路由、瘦身、缓存、批处理、限额与可观测性，让单位业务价值对应的 AI 花费可控、可预测、可优化**。

回扣故事：

- **「费用翻三倍、用户只涨四成」** → 典型 **非线性成本**：Agent 多跳、上下文变厚、默认旗舰模型；  
- **「一条会话十二趟」** → **全链路计费**：不只主模型，还有 Embedding、Rerank、Guardrails、重试；  
- **「分车型、减重量、别空车、批处理」** → **模型路由、上下文裁剪、Prompt/结果缓存、离线批处理**；  
- **PRD 加「单次成本」列** → 产品要把 **成本作为一等需求**，与准确率、时延并列；  
- **「Demo 免费、量产按斤卖」** → 上线前要用 **真实流量形态** 做 **成本压测**，不能只看实验室 Demo。

在 **AI 产品岗** 的典型场景：

| 场景 | 你要怎么用这个概念 |
|------|-------------------|
| **需求文档** | 每个 AI 功能写清 **目标单次成本区间**、**允许调用的模型档位**、**最大轮次/最大上下文**；复杂 Agent 附 **熔断与转人工** 条件 |
| **方案评审** | 问：「这条链路 **一共调几次模型**？ **Tool 返回裁不裁剪**？有没有 **缓存命中**？高峰 **限流策略** 是什么？」 |
| **指标设计** | 除准确率、时延外，增加 **分场景单次成本、成本异常率、AI 解决单 vs 人工单成本对比**；降本改动 **必须挂评测回归** |
| **跨团队沟通** | 跟财务说：「 **总账单要拆到场景**，否则不知道砍哪」；跟算法说：「 **路由规则要可配置、可 A/B**」；跟业务说：「 **免费深度对话有上限**，是 **商业模型** 不是技术抠门」 |

## 类比锚定

**核心类比：电商大促前的「物流与包材成本管控」——传统 PM 熟 **按单履约成本、分仓、选快递档位、热销品预打包、超区加价**；推理成本是把同一套 **「别用空运寄明信片」** 思维搬到 AI：**按次计费、分模型档位、减包裹重量、热点缓存、高峰限单**。**

| 你熟悉的 PM 经验 | Inference Cost 要素 |
|------------------|---------------------|
| 快递 **分档位**：同城标快 vs 次日达 vs 航空 | **模型路由**：小模型处理 FAQ，旗舰处理高风险/复杂推理 |
| 包裹 **减重**：去掉多余填充、合并发货 | **上下文瘦身**：裁剪 Tool 返回、检索 Top-K 控制、历史摘要 |
| 爆款 **提前入前置仓**，少走长途 | **Prompt / Embedding 缓存**：固定前缀、重复问法命中缓存 |
| 大促 **夜间批量出库**，白天留通道给急单 | **批处理推理**：离线评测、标注、摘要走低价时段/批处理 API |
| 包邮门槛与 **单笔订单毛利** 对齐 | **产品分层**：免费额度、深度功能付费、超轮次转人工 |
| 看 **每单履约成本** 而不只看 GMV | 看 **单次有效咨询 AI 成本** 并对标 **人工坐席成本** |

## 类比边界说明

### 一、概念核心要点（回扣定义）

1. **推理成本是「全链路乘法」**：主模型只是一环； **检索、重排、多轮 Agent、复检、失败重试** 都会乘上去——产品要看 **调用链账单**，不是单看报价单上的「每千 Token 单价」。
2. **降本必须绑质量底线**：路由、裁剪、限额都要 **挂评测与在线指标**； **偷偷换小模型** 是最危险的「省钱」。
3. **成本是产品策略，不只是工程优化**： **免费额度、功能分层、转人工时机** 和 **商业模型** 一体设计——不是研发单方面「调参抠钱」。
4. **Demo 与量产的成本形态完全不同**：实验室 **短对话、低并发** 会严重 **低估** 真实账单；上线前要做 **成本压测与异常熔断**（防 Agent 死循环）。
5. **可观测性先于优化**：先能 **按场景拆账、抽样高成本会话**，再谈砍哪一刀；否则 **砍错地方伤体验，砍对地方也不知道**。

### 二、这个类比在哪里会失效？

| 类比说法 | 失效点 | 更准确的理解 |
|----------|--------|--------------|
| 「像物流，越省越好」 | 过度降本 **直接伤害准确率、合规、品牌**；有些场景 **必须贵**（合同、医疗、金融） | **分场景成本上限** + **质量门禁**；贵是 **设计选择**，不是失误 |
| 「选小快递等于选小模型」 | 模型能力边界 **非线性**；小模型在 **推理、多步、长尾** 上可能 **断崖式变差** | 用 **金标集分场景验证** 再路由；保留 **升舱兜底**（答不好再上大模型） |
| 「缓存像前置仓，一劳永逸」 | 政策、价格、知识库 **频繁变**；缓存 **过期即错答 + 合规风险** | 缓存要 **设 TTL、版本号、失效策略**；高频变内容 **不宜长缓存** |
| 「按件计费，件数看得见」 | Token 与 **语种、格式、JSON 冗长** 相关；同样「一句话」 **Token 数可差几倍** | 成本估算看 **实测 Token**，不看 **用户字数**；Tool 返回 **结构化裁剪** |
| 「限流像快递停发」 | 粗暴限流 **直接伤害转化与体验**；用户感知为 **「AI 不好用」** 而非 **「系统在省钱」** | 限额要 **产品化表达**（额度、升级、排队、转人工）； **透明 + 可预期** |

### 动手小练习（约 15 分钟）

1. 选你负责的一个 AI 功能，画 **调用链**：从用户发送到返回， **数清几次模型/API 调用**；猜哪一环 **最贵**（不必精确到分）。
2. 写一张 **三行成本表**：场景名、 **目标单次成本**、 **超支时的兜底**（限轮次 / 转人工 / 禁旗舰）。
3. 打开任意模型 **定价页**，用 **「输入 8K + 输出 2K Token」** 粗算一笔单次对话费用，再乘以 **你业务的日会话量**——看是否 **接近财务现实**。
