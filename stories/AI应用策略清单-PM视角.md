# AI 应用策略清单（PM 视角）

| 字段 | 内容 |
|------|------|
| **用途** | 传统 B 端 PM 做 AI 应用时的 **规则 / 策略层检查表**（可贴进 PRD） |
| **难度** | 基础偏应用 · 上线专题 |
| **配套** | [系列世界观](世界观.md) · [故事索引见仓库 README](../README.md) |
| **原则** | 模型给概率，**产品用策略兑现承诺**；每条策略写清 **默认值、谁可改、如何观测、失败退路** |

> **怎么用**：中小团队先勾「P0」；有对外入口、Agent 高危工具、强监管时再勾「标准增强」。  
> **对照故事**：括号内为系列篇号，便于回看寓言场景。

---

## 0. 总原则（写进 PRD 首页）

- [ ] 本产品 **不可完全信任模型输出**；关键结论需闸门或人工
- [ ] 策略分四层：**路由 / 安全权限 / 质量闸门 / 成本体验**（可交叉）
- [ ] 每条策略有：**默认行为、配置入口、日志字段、回滚开关**
- [ ] 标明 **标准规范流程** vs **中小默认方案**，以及升级触发条件（流量、监管、对外、高危工具）

## 7. 按场景的最小策略包（可直接复制）

### 7.1 企业内部知识问答（如内小言）

P0：System Prompt · RAG 权限 · 混合检索分路 · Grounding · 灰区先拦 · 申诉值班  
暂缓：即时人审、全量记忆向量混检、旗舰模型默认开

### 7.2 售后拍照报修（如修小言）

P0：多模态路由 · 拍前引导 · 低置信补拍 · 涉电转人工 · 工具/工单字段校验 · 成本上限  
暂缓：自研视觉分类大盘（路线 B 全集）、全员旗舰 Vision

### 7.3 Agent 调业务系统（导出/入账）

P0：工具白名单 · 参数与租户校验 · 高危二次确认 · 结构化 Schema · 审计日志  
暂缓：开放任意网页工具、模型自由拼 SQL

## 8. 一页纸自检（发版前 10 问）

1. 用户从哪个入口进来？意图错了会落到哪条退路？  
2. 本轮会动用哪些模型/检索/工具？各阈值是什么？  
3. 哪类输出 **禁止** 直接展示或写入系统？  
4. 低置信 / 无出处 / Schema 失败时，用户看到什么？  
5. 跨用户、跨应用、跨租户是否硬隔离？  
6. 灰区是先拦后放，还是等审？超时默认是什么？  
7. 单次成本上限？超了如何降级？  
8. 日志能否按 trace 还原「用了哪条策略」？  
9. 红队用例是否覆盖：注入、套话、越权、串味、模糊铭牌？  
10. 策略变更谁审批、如何回滚？
